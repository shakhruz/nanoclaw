# Video Analysis

Универсальный пайплайн для анализа видео. Работает локально в контейнере через `ffmpeg` + `yt-dlp` + `faster-whisper`. Кадры отдаются Claude как multi-image input, транскрипт — текстом в том же сообщении. Результат: что человек говорит + что показано в кадре + ключевые моменты с timestamps.

## When to invoke

- Пользователь форвардит/прикладывает видео (mp4, mov, webm) или **круглое видеосообщение** Telegram (`video_note`).
- Пользователь даёт ссылку на Instagram Reel / YouTube / TikTok / Telegram-пост с видео и просит «посмотри», «что там», «проанализируй».
- Просьба «расшифруй» (только аудио) — тоже сюда: пайплайн отработает без кадров (`--no-frames`).
- Любой видео-контент клиента, который нужно подытожить, найти моменты, оценить визуал/тон.

**Не вызывать** если:
- видео > 2 часов (предупреди пользователя, спроси можно ли обрезать или предложи ему YouTube-ссылку с конкретным timestamp);
- это GIF без аудио — используй обычный image-анализ через Read.

## Inputs

| Источник | Что передавать в `--input` |
|---|---|
| Прикреплённый файл из Telegram | абсолютный путь, который пришёл в `image_path`/`attachment_file_id` (для file_id сначала `download_attachment`) |
| Ссылка YouTube/Instagram/TikTok/Telegram | URL целиком |
| Локальный файл из `/workspace` | путь |

## Pipeline

```bash
/app/skills/video-analysis/analyze.sh --input <PATH_OR_URL> [options]
```

Скрипт делает:

1. **Получение файла**:
   - URL → `yt-dlp -f "best[height<=1080]" -o /tmp/va_<hash>.%(ext)s <url>`
   - Локальный файл → используется напрямую.

2. **Memoize**: считает sha256, проверяет `/workspace/group/cache/video-analysis/<hash>.json`. Если есть — возвращает кэш мгновенно. Кэш живёт 30 дней.

3. **Метадата** через `ffprobe` → длительность, fps, разрешение, наличие аудиодорожки, aspect ratio (для круглого видеосообщения соотношение ~1:1, длительность всегда ≤ 60с).

4. **Frame sampling** — стратегия выбирается по длительности:

   | Длительность | Стратегия | Макс кадров |
   |---|---|---|
   | < 30 сек | каждый кадр через `fps=1` | 30 |
   | 30 сек – 5 мин | I-frames (`-vf "select='eq(pict_type,I)'"`) | 30 |
   | 5 мин – 30 мин | I-frames + 1 кадр / 30 сек | 40 |
   | > 30 мин | 2-pass: транскрибируется аудио → агент в Bash вызывает `analyze.sh --extract-frames-at "<csv timestamps>"` для интересных моментов | 60 |

   Все кадры даунскейлятся до max-1024px (сторона), JPEG q=85, в `/tmp/va_<hash>_frames/frame_<seconds>.jpg`.

5. **Audio**: `ffmpeg -i <video> -vn -ac 1 -ar 16000 -f wav /tmp/va_<hash>.wav` → `faster-whisper` на этом WAV. Язык auto-detect, но для русского принудительно `--language ru` если в контексте чата русская речь.

6. **Output JSON**:
   ```json
   {
     "duration_sec": 47.3,
     "resolution": "1080x1920",
     "is_round_note": false,
     "frame_count": 12,
     "frames_dir": "/tmp/va_abc/frames",
     "transcript": {
       "text": "Привет! Сегодня я расскажу...",
       "segments": [
         {"start": 0.0, "end": 4.2, "text": "Привет!"},
         ...
       ],
       "language": "ru"
     },
     "cached": false
   }
   ```

## После того как скрипт отработал

1. Прочитай JSON.
2. Если `frames_dir` есть — собери все кадры через Read (как изображения) и пошли Claude в одном сообщении вместе с транскриптом.
3. Сделай **structured summary** в формате:
   - **О чём (1-2 фразы):** …
   - **Транскрипт ключевых фраз** с timestamps (если ролик > 30 сек): `[00:12]` … `[00:34]` …
   - **Что показано** (визуал): сцены, люди, объекты, эмоции, цвет/свет/композиция если важно
   - **Главные моменты** (для длинных): 3-5 пунктов с timestamps
   - **Тон/настроение** (если применимо к контенту, например для рилсов клиента)
4. Если пользователь спросил конкретное (например «что говорит про цены») — отвечай на это.

## Round video notes (`video_note`)

Telegram-«кружочки»: всегда квадрат, ≤60 сек, ≤4 MB. Адаптер сохраняет как обычный mp4 в attachments. Пайплайн работает as-is. Просто обычно достаточно `1 fps` стратегии и 5-10 кадров — а транскрипт самое ценное.

## Cost notes

- 30-сек ролик: ~10 кадров + ~$0.005 транскрипта = **~$0.02–0.05** за вызов
- 5-мин: ~30 кадров + ~$0.05 = **~$0.10–0.15**
- 30-мин: keyframes (~40) + ~$0.30 = **~$0.30–0.50**
- 2-часовой: 2-pass — сначала только транскрипт ($1.20), потом кадры из ~5 интересных моментов = **~$1.50** (без 2-pass было бы $5+)

## Options reference

| Flag | Default | Описание |
|---|---|---|
| `--input <path\|url>` | required | Источник видео |
| `--language <ru\|en\|auto>` | `auto` | Язык для whisper. `auto` — автодетект. |
| `--no-frames` | off | Только транскрипт, без кадров (для подкастов / только-аудио) |
| `--whisper-model <small\|medium\|large-v3>` | `$WHISPER_MODEL` (default `small`) | Модель whisper. `medium` для шумного аудио. |
| `--max-frames <N>` | по таблице выше | Жёсткий потолок кадров |
| `--extract-frames-at <csv>` | — | 2-pass: вытащить кадры на конкретных timestamps в секундах, например `12.5,34,67.2`. Использовать в комбе с `--no-transcribe`. |
| `--no-transcribe` | off | Пропустить аудио (используется во 2-passе) |
| `--no-cache` | off | Игнорировать кэш |
| `--cache-dir <dir>` | `/workspace/group/cache/video-analysis` | Где хранить memoize |
| `--output <path>` | stdout | JSON output |

## Failure modes

- `yt-dlp` 403 / private video → сообщи пользователю «не могу скачать, дай файл».
- whisper падает на тишине → транскрипт = `""`, не блокирующая ошибка.
- Видео > 2 часа → предупреди пользователя и спроси разрешения (запустится дороже).
- Нет аудиодорожки → транскрипт пустой, только кадры.

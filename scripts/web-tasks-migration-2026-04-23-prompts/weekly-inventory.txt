Weekly web-projects inventory refresh (cron 20:00 Sunday Tashkent).

Цель — обновить `/workspace/global/web-projects/inventory.json`, выявить orphans (Vercel без GitHub, GitHub без Vercel, домен без деплоя).

Алгоритм:

1. Запусти `/home/node/.claude/skills/web/refresh-inventory.sh` — это перегенерирует `inventory.json`.

2. Загрузи свежий `inventory.json` и посчитай:
   - Всего Vercel-проектов
   - Всего кастом-доменов
   - Всего локальных клонов в `~/apps/`
   - **Orphans**:
     - `vercel_project`, который не упоминается ни в `local_clones`, ни в `github_repos` по имени (возможно legacy-деплой)
     - `github_repo`, который не упомянут ни в одном Vercel-проекте по имени (репо без деплоя)
     - Домен из `vercel_domains`, которого нет в `vercel_projects[].vercel_prod_url` (домен без активного проекта)

3. Сформируй краткий отчёт в main (через `mcp__nanoclaw__send_message`):

```
📦 *Weekly Web Inventory*

✅ Активно: {N} Vercel-проектов / {M} доменов / {L} клонов
⚠️ Orphan Vercel: {count} (например: {first-3})
⚠️ Orphan GitHub: {count} (например: {first-3})

Последнее обновление: {timestamp}
```

4. Если orphan'ов больше 5 — сохрани полный список в `/workspace/global/web-projects/audits/YYYY-MM-DD-weekly-orphans.md` и пришли ссылку.

5. **Не** пытайся сам чистить orphan'ы. Только сводка.

Никаких браузер-вызовов, никаких деплоев.
